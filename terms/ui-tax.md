# UI Tax

**Canonical definition (Arco Lexicon)**
> The cost premium embedded in software products designed for human cognition — including graphical interface development, permissions infrastructure, user training, and the subscription markup charged for enterprise UX — which is entirely redundant in an agentic stack where agents require no interface to operate.

## Extended Definition

The UI Tax is not a surcharge added on top of a software product's core value. It is the core of the product's cost structure. SaaS vendors price primarily for the interface layer: the dashboard a manager uses to review outputs, the permissions system that controls who can access what, the onboarding flow that trains new users, the enterprise support tier staffed by humans who explain how to use the GUI. Every element of the per-seat price exists to serve a human who needs to see, click, configure, and understand.
An autonomous agent operates on none of these signals. It calls an API endpoint and receives a structured response. It does not log in. It does not navigate a dashboard. It does not require onboarding or support. The graphical layer, the access controls, the user training resources — these are built for human operators and priced accordingly. In an agentic workflow, they are dead weight in the cost base: a permanent overhead on every process the agent runs, for capabilities the agent will never use.
